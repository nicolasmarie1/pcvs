pcvs.testing.tedesc module

class pcvs.testing.tedesc.TEDescriptor(name, nodecontent, label, subprefix)[source]

Bases: object

A Test Descriptor (named TD, TE or TED), maps a test program representation, as defined by a root node in a single test files.

A TE Descriptor is not a test but a definition of a program (how to use it, to compile it…), leading to a collection once combined with a profile (providing on which MPI processes to run it, for instance).

Variables:

  • _te_name (str) – YAML root node name, part of its unique id

  • _te_label (str) – which user directory this TE is coming from

  • _te_subtree (str or NoneType) – subprefix, relative to label, where this TE is located

  • _full_name (str) – fully-qualified te-name

  • _srcdir (str) – absolute path pointing to the YAML testfile dirname

  • _buildir (str) – absolute path pointing to build equivalent of _srcdir

  • _skipped (bool) – flag if this TE should be unfolded to tests or not

  • _effective_cnt (int) – number of tests created by this single TE

  • _program_criterion (Criterion) – extra criterion defined by the TE

  • others – used yaml node references.

Constructor method.

Parameters:

  • name (str) – the TE name

  • node – the TE YAML content.

  • label (str) – the user dir label.

  • subprefix (str) – relative path between user dir & current TE testfile

Raises:

TDFormatError – Unproper YAML TE format (sanity check)

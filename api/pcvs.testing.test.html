pcvs.testing.test module

class pcvs.testing.test.Test(comb=None, wd=None, resources=None, environment=None, te_name='noname', label='nolabel', subtree='nosubtree', user_suffix=None, command='', metrics={}, tags=[], artifacts={}, validation={}, mod_deps=[], job_deps=[])[source]

Bases: object

Smallest component of a validation process.

A test is basically a shell command to run. Depending on its post-execution status, a success or a failure can be determined. To handle such component in a convenient way, more information can be attached to the command like a name, the elapsed time, the output, etc.

Variables:

  • NOSTART_STR (str) – constant, setting default output when job cannot be run.

  • DISCARDED_STR (str) – constant, setting default output for discarded test.

Construct a Test.

Parameters:

  • comb (Combination | None) – Criterion expanded combination for this specific test.

  • wd (str | None) – Test working directory.

  • resources (list[int] | None) – Resources consumed by the test (cpu_cores / nodes), scheduler heuristic to avoid running to much jobs in scheduler that badly scale (like slurm).

  • environment (list[str] | None) – Environment variables used when running the test.

  • te_name (str) – Test name.

  • label (str) – Test label.

  • subtree (str) – Sub directory of the test from the pcvs execution directory.

  • user_suffix (str | None) – Test suffix.

  • command (str) – The command used to run the test.

  • metrics (dict[str, dict[str, Any]]) – Dictionary of regexs to get data from test output, stored in test results.

  • tags (list[str]) – Tets tags, used to filter test execution and console output of test stdout/stderr.

  • artifacts (dict) – Artifacts (files) that will be saved in test data after test run, a dict mapping artifact name in test data to files names to read from the disks.

  • validation (dict) –

    Validation configuration. Contain some to all of the following:

    • expected exit code

    • soft_timeout

    • hard_timeout

    • dynamic time validation ((mean + tolerance) * weight)

    • regex matcher

    • validation script path

  • mod_deps (list[PManager]) – Package manager dependency (modules or spack).

  • job_deps (list[str]) – Other jobs name that this test depends on (like compilation phases).

property needed_resources: list[int]

Return the orcherstrator resources used by the jobs

The meaning for resources list is user defined and can vary depending on how test/plugin defines the resources and how the job orcherstrator of the wrapper defines themes.

It will most likely be [nb_nodes, nb_cpu_per_nodes]. But it could be [nb_nodes_lvl1, nb_nodes_lvl2, nb_nodes_lvl3, NUMA_NODE, UNIX_process, MPI_PROCESS, L3_CACHE, pthreads, L2_CACHE, omp_threads, L1_CACHE, …].
